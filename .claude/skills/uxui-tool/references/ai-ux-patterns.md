# AI/ML UX Patterns

Evaluate AI interface patterns including explainability, trust signals, error handling, and user control.

## AI Transparency Principles

### Core Requirements

| Principle | Description | Implementation |
|-----------|-------------|----------------|
| **Disclosure** | User knows AI is involved | Label AI-generated content |
| **Explainability** | User understands why | Show reasoning/factors |
| **Control** | User can override | Edit, reject, feedback |
| **Feedback** | User can improve | Thumbs up/down, corrections |
| **Accountability** | Clear responsibility | Human review for high-stakes |

## Explainability Patterns

### Explanation Levels

| Level | Audience | Content |
|-------|----------|---------|
| Basic | All users | "Based on your activity" |
| Detailed | Curious users | "These 3 factors influenced..." |
| Technical | Power users | Model details, confidence % |

| Pattern | Use Case | Example |

| **Expandable** | Optional detail | "Why this?" link |
| **Tooltip** | Non-intrusive | Hover for explanation |
| **Side panel** | Deep dive | Detailed breakdown |
| **Confidence meter** | Uncertainty | "85% confidence" |

### Explainability Checklist

## Error Handling for AI

### AI-Specific Error Types

| Error Type | User Message | Recovery |
|------------|--------------|----------|
| Low confidence | "I'm not sure about this" | Offer alternatives |
| No result | "I couldn't find anything" | Suggest retry/manual |
| Timeout | "Taking longer than expected" | Progress + cancel |
| Rate limit | "Please wait before trying again" | Clear timing |
| Model failure | "Something went wrong" | Retry + fallback |

### Error Message Pattern

```
[What happened] + [Why (if known)] + [What to do]
```
